个人博客搭建过程

1、安装npm

2、安装cnpm

3、安装hexo

输入命令 cnpm install -g hexo-cli

4、在git cmd窗口内克隆hexo

直接在windows的cmd窗口内输入命令不好用，要在git中输入命令 hexo init，等待一会儿

5、到这一步就搭建好了，启动网站页面

输入命令 hexo server，弹出网站页面如下

6.新建一篇文章

ctrl+c再输Y断开批处理操作

hexo n “我的第一篇博客文章”

输入命令hexo g 生成

再次启动得到：

7、部署到远端【搞了好半天】

准备工作：首先，在github上部署，新建一个仓库，再安装git部署插件输入命令：npm install –save hexo-deployer-git；

再设置myWeb文件夹下的_config.yml文件，windows系统直接用notepad++打开，在最下边添加deploy信息【每个冒号后边要加空格！！】

下一步，先打开git Bash，输入如下命令（给git配置身份）：

git config --global user.name "xxx" 其中xxx为github的用户名

git config --global user.email "xxx" 其中xxx为github的登录邮箱

接下来回到git CMD中，输入：

hexo d

这里的弹出一个选项：

我选择1，然后弹出一个网页，是git请求绑定github的请求，允许之后回到cmd窗口等待片刻后弹出如下信息后即部署成功！

此时刷新github的仓库，发现已经部署好了！（开心！！！）

现在只要在浏览器直接输入仓库名就可以访问啦！

8、自定义更换主题

https://github.com/litten/hexo-theme-yilia github上一位大佬做的主题，名字为yilia

以该主题为例，命令行输入

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

等待一会儿克隆完成，此时themes文件夹中有了yilia这个文件夹

再次修改config.yml文件，将theme改为yilia

然后清理一下，重新生成，最后打开，输入如下命令：

hexo clean

hexo g

hexo server

注意：这个时候还没有推到远端，还是要用localhost：4000来访问网站

然后再输入hexo d推到远端，就可以用仓库名直接访问了，主题更换完成~

1.8 IE缓存问题

IE浏览器会对ajax的请求结果做一个缓存，下次发送请求时，走的是本地缓存而不是服务器返回的最新数据，对于时效性强的场景，会影响结果。

IE在代码更改时，如果不刷新页面，发送请求得到的结果也不会更新，即【不支持ajax的实时重新加载】

1.10 重复请求问题

当用户疯狂点击，反复请求，服务器响应又慢时，服务器压力会很大。

解决办法：发送请求后看看之前有没有相同请求，如果有就取消原来的请求，留下后一请求，这样同种请求同时只会有一个。设置一个标识变量，意义为判断是否正在发送请求，初始值为false；发送时设为true，请求完成时还原成false。

第三章 跨域

3.1同源策略

同源策略是浏览器的一种安全策略。网页的URL和AJAX请求的目标资源的URL的【协议】、【域名】、【端口号】必须完全相同。AJAX默认遵守同源策略。不满足同源策略不能直接发AJAX请求。

违背同源策略就是【跨域】。

3.2.2 CORS

  1. CORS是什么？

    CORS（cross-origin resource Sharing）,跨域资源共享。官方的跨域解决方案，特点是不需要在客户端做任何特殊操作，完全在服务器中进行处理，支持get和post请求。跨域共享标准新增了一组http字段

  2. CORS怎么工作？

    通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该相应以后就会响应放行。

在服务端添加一个setHeader就可以啦！
