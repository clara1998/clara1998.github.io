Node.js学习笔记

1、命令行窗口（CMD窗口、终端、SHELL）

  • 常用指令：

    • dir：理出当前，目录下的所有文件
    • cd 目录名：进入到指定的目录
    • md 目录名：新建一个文件夹
    • rd 目录名：删除一个文件夹
    • 直接输文件名： 打开这个文件
    • 换别的盘就直接 D：

  • 目录：

    • . 表示当前目录
    • .. 表示上一级目录（../style.css,当前文件实际上是省略了./）

  • 环境变量（windows系统中的变量）

    一般在环境变量中修改

    • path：把某个文件的路径加入path后重启命令行窗口,就可以在任何位置打开文件了

    • 当我们在命令行窗口打开一个文件，或者调用一个程序时，系统会首先在当前目录下寻找文件程序，如果找带了直接打开，如果没找到则会依次到环境变量path的路径中寻找，直到找到为止。如果没找到，则报错

    • 类似于作用域链

    • 所以可以将一些经常需要访问的程序和文件的路径添加到paht中，这样我们就可以在任意位置来访问了

2、进程和线程

  • 进程

    • 进程负责为程序的 运行提供必备的环境
    • 进程就相当于工厂中的车间（看CPU，占用为0的说明没有在工作）

  • 线程

    • 线程是计算机中的最小的计算单位，线程负责执行进程中的程序
    • 线程相当于工厂中的工人

  • 单线程（一个活一个人干）

    • js是单线程。在执行js代码时，网页的渲染是停止的，执行完渲染

  • 多线程（一个活多个人干）

    • 假设一个线程加载页面，一个渲染页面，加载到一半js代码里改样式，就会矛盾

3、node.js简介

node.js是一个能在服务器端运行js的js运行环境。（之前都是在浏览器端运行，node.js可以直接在系统中运行，平台不仅限于浏览器了，可以像java等语言一样直接和系统来交互）

把js的战场从前端拉到服务器

  • I/O：对磁盘的读写操作

Node

  • node师弟ES标准的一个实现，也是一个js引擎
  • 通过node可以使js代码在服务器端执行
  • node仅仅对ES标准进行了实现，所以在node中不包含ＤＯＭ和ＢＯＭ
  • node中可以使用所有的内联对象
    • String Number Boolean Math Data RegExp Function Object Array
    • 而DOM和BOM都不能使用，但是可以使用console也可以使用定时器（setTimeout（） setInterval（））
  • node可以再后台来编写服务器
    • node编写服务器都是单线程的服务器
    • 进程
      • 进程负责为程序的 运行提供必备的环境
      • 进程就相当于工厂中的车间（看CPU，占用为0的说明没有在工作）
    • 线程
      • 线程是计算机中的最小的计算单位，线程负责执行进程中的程序
      • 线程相当于工厂中的工人
  • 传统的服务器都是多线程的
    • 每进来一个请求，就创建一个线程去处理请求
  • node的服务器是单线程的
    • node处理请求时是单线程，但是在后台拥有一个I/O线程池
  • 分布式：一个服务器不够，再加一个

4、包

1.包的结构

包就是一个压缩文件，解压以后还原为目录

  • package.json 描述文件【必须！】

  • bin 可执行二进制文件

  • lib js代码

  • doc 文档

  • test 单元测试

包描述文件勇于表达非代码相关的信息，位于包的根目录下

5、NPM（Node Package Manager）

npm -v 查看版本

npm version 看所有模块版本

npm search 包名 查找包（需要联网）

npm install 包名 在当前目录安装包 或者npm i 包名

npm install 包名 -g 全局模式安装包（全局安装的包一般都是一些工具，在计算机利用，不是在项目里用）

npm install 包名 –save 安装包并添加到依赖中【常用！】

npm install 下载当前项目所依赖的包

8、fs模块

同步和异步调用：

  • fs模块中所有操作都有两种形式：同步和异步
  • 同步文件系统会阻塞程序的进行，即除非操作完毕，否则不会向下执行代码
  • 一部文件系统不会阻塞程序的进行，而是在操作完成时，通过回调函数将结果返回
同步文件写入：

文件系统（File System）
- 文件系统简单来说就是通过node来操作系统中的文件
- 使用文件系统，需要引入fs模块，fs是核心模块，直接引入不需要下载
同步文件的写入
- 手动操作的步骤
1.打开文件
fs.openSync(path,flags[,mode])
- path 要打开文件的路径
- flags 打开文件要做的操作类型
r 只读的
w 可写的
- mode 设置文件的操作权限，一般不传
返回值：
- 该方法会返回一个文件的描述符作为结果，我们可以通过该描述符来对文件进行各种操作
2.向文件中写入内容
fs.writeSync(fd,string[,position[,encoding]])
- fd 文件的描述符，传递我们要写入的文件的描述符
- string 要写入的内容
- position 写入起始位置
- encoding 写入的编码
3.保存并关闭文件
fs.closeSync(fd)
- fd 要关闭的文件的描述符

异步文件写入：
fs.open(path,flags[,mode],callback);

- 用来打开一个文件
    - 异步调用的方法，结果都是通过回调函数的参数返回的
    - 回调函数两个参数：
        err 错误对象，如果没有错误则为null
        fd 文件的描述符
    fs.write(fd,string[,position[,encoding]],callback)
    - 用来异步写入一个文件
    fs.close(fd,callback)
    - 用来关闭文件
